# Структура Проекта

Приложение представляет собой Windows-only оверлей-лупу на базе нативного Magnification API. Оно не использует инжект в процесс игры и держит оверлей в отдельном прозрачном topmost-окне.

## Модули

`src/main.py`
Запускает приложение и передаёт управление точке входа пакета.

`src/lens_overlay/app.py`
Создаёт runtime-объекты, загружает конфигурацию, инициализирует оверлей и запускает главный цикл Win32.

`src/lens_overlay/config.py`
Загружает и сохраняет `settings.json`. Хранит редактируемые настройки времени выполнения:
- пресеты масштаба и текущий масштаб
- текущий квадратный размер и пределы изменения размера
- позицию лупы
- режим активации
- форму
- политику частоты обновления (`refresh_hz`, где `0` означает режим без ограничения)
- привязки клавиш и мыши

`src/lens_overlay/models.py`
Небольшие общие типы данных проекта:
- `Point`
- `Size`
- `Rect`
- `ActivationMode`
- `LensShape`

`src/lens_overlay/win32.py`
Тонкая обёртка над нужными частями WinAPI через `ctypes`:
- создание окон и обработка сообщений
- опрос состояния клавиатуры и мыши
- создание регионов окна для формы лупы
- вызовы Magnification API

`src/lens_overlay/overlay.py`
Владеет прозрачным окном-хостом и дочерним контролом magnifier.

Основные задачи:
- инициализация и завершение Magnification API
- создание topmost click-through окна оверлея
- создание magnifier-контрола
- применение формы окна
- обновление границ оверлея
- установка матрицы масштаба
- установка исходного прямоугольника увеличения

`src/lens_overlay/input.py`
Опрос состояния клавиатуры и мыши без хуков и без инжекта.

Основные задачи:
- сопоставление логических действий виртуальным клавишам
- построение `InputSnapshot`
- определение фронтов нажатия и отпускания (`pressed`, `released`) для runtime-действий

`src/lens_overlay/controller.py`
Центральная логика лупы.

Основные задачи:
- поведение активации
- переключение пресетов масштаба
- увеличение и уменьшение размера
- перемещение с клавиатуры
- перетаскивание мышью
- ограничение позиции границами виртуального экрана
- преобразование позиции и размера лупы в исходный прямоугольник для увеличения
- периодическое сохранение конфигурации

## Поток Данных

1. `app.py` загружает `settings.json` через `ConfigStore`.
2. `OverlayWindow` создаёт прозрачное окно-хост и контрол Magnifier.
3. `InputManager` каждый кадр опрашивает глобальное состояние клавиатуры и мыши.
4. `LensController` читает ввод и меняет состояние конфигурации:
   - активно или неактивно
   - текущий пресет масштаба
   - текущий квадратный размер
   - текущая позиция на экране
5. `LensController` вычисляет:
   - границы оверлея на экране
   - исходный прямоугольник, который захватывает Magnification API
6. `OverlayWindow` применяет новые границы, матрицу масштаба и источник.
7. `ConfigStore` периодически записывает обновлённые настройки обратно в `settings.json`.

## Основное Поведение Во Время Работы

Режим активации по умолчанию: `hold` на `Left Alt`.

Управление по умолчанию:
- `Left Alt`: показывать лупу, пока клавиша удерживается
- `Ctrl + Left Alt + Arrow keys`: перемещать лупу
- `Ctrl + Left Alt + Plus/Minus`: переключать пресет масштаба
- `Ctrl + Left Alt + Page Up/Page Down`: менять размер лупы на 1 пиксель
- `Ctrl + Middle Mouse Drag`: перетаскивать лупу
- `Ctrl + Left Alt + End`: выход из приложения

Эти привязки намеренно сосредоточены в конфигурации, чтобы поведение можно было менять без переработки кода оверлея или контроллера.

## Точки Расширения

Проект разложен так, чтобы дальнейшая доработка оставалась локальной:

Варианты формы
- Добавить новые значения в `LensShape`
- Расширить создание региона окна в `overlay.py`

Рамка или обводка
- Добавить настройки рамки в `config.py`
- Рисовать inset-окно magnifier или отдельный слой рамки в `overlay.py`

Плавность движения или анимация
- Хранить целевую позицию в `controller.py`
- Интерполировать состояние перед вызовом `overlay.update_bounds`

Профили настроек
- Расширить `settings.json`
- Добавить выбор профиля в `config.py`

Альтернативные режимы активации
- Добавить новый режим в `ActivationMode`
- Реализовать переход состояния в `controller.py`

Альтернативные backend-ы захвата
- Сохранить контракт контроллера без изменений
- Заменить или дополнить `overlay.py` и `win32.py`, не трогая модули конфигурации и ввода
